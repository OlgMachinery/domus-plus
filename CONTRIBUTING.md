# Contribuir a DOMUS+

Gracias por querer contribuir. Aquí un flujo simple para que el equipo colabore de forma ordenada.

1. Fork / Clone
   - Fork del repo si trabajas desde cuenta personal, o clona directamente si tienes acceso.

2. Ramas
   - Crea ramas descriptivas: `feature/<breve-descripcion>`, `fix/<issue-#>`

3. Commits
   - Commits claros y pequeños.
   - Sigue un formato: `feat`, `fix`, `chore`, `docs`, `refactor`.

4. Pull Request
   - Abre PR contra `main` o `develop`.
   - Incluye descripción, pasos para reproducir y archivos cambiados.

5. Revisiones
   - Asigna revisores del equipo.
   - Resuelve comentarios antes de merge.

6. Tests & CI
   - Asegúrate de que el build corra localmente (`npm run build`) antes de PR.
